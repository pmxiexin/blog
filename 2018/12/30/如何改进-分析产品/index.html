如何改进&分析产品

字数统计: 2.4k | 阅读时长 ≈ 7

产品系统分析

通过产品内在分析来寻找优化点。有以下两种分析思路。

树状分析法

通过从用户出发，分析用户可能碰到的任何情况和需求和解决方案。
分为三个阶段：目标分析、需求分析 和 方案分析。

目标分析

分析所需目标，明确目的和优先级

  1. 与上级商议期望目标内容
  2. 罗列所有期望目标
  3. 将目标进行关联，如拉新类、增长类、营收类、留存类，并在这基础上进行同级联想和下级扩展
  4. 获得树状目标分析图，以”总目标 - 大目标 - 小目标“的格式呈现，如”提高营收 - 提高大R营收 - 增强比赛竞猜氛围“
  5. 将目标进行优先级排序

需求分析

  1. 根据“用户 - 特征 - 场景 - 心理 - 需求”来处理完成需求树状结构分析图。并选取其中优先级更高的需求，进入方案分析阶段。如“家庭主妇 - 负责家庭开销 - 网上购物 - 省钱 - 找到同一商品价格最低的“。
  2. 分析内容：
    1. 用户：当前产品主要存在的用户分类，如“家庭主妇”、“学生”
    2. 特征：该用户所存在的主要特征，如“家庭主妇需操劳家庭日常开销”
    3. 场景：该特征主要发生的场景，含线上/线下。如“搜索商品时”等
    4. 心理：该场景所引发出的心理情况，如“想要省钱”、“担心麻烦”等
    5. 需求：在该场景、该心理下用户的需求内容
  3. 分析结果：
    1. 完成需求树状分析图
    2. 对需求进行优先级排序

方案分析

  1. 选取重要的需求进行方案分析，有以下两种分析路径
    1. 基于目标：根据”需求 - 目标 - 解决方案 “完成方案树状结构分析图。如”找到同一商品价格最低的 - 促进用户购买 - 高亮价格最低的商品“
    2. 基于问题：根据”需求 - 当前用户行为 - 当前存在的问题 - 解决方案“完成方案树状结构分析图。如”找到同一商品价格最低的 - 用户通过价格降序 - 价格最低的不一定靠谱 - 价格最低的基础上加上系统推荐“
  2. 分析内容
    1. 目标：针对用户的需求可实现的目标
    2. 当前用户行为：在目前的情况下，当前用户会怎么满足需求
    3. 当前存在的问题：用户当前满足需求的方式中的不方便之处
    4. 解决方案：当前用户针对该问题的处理方案
  3. 方案结果：
    1. 完成方案树状分析图
    2. 对方案进行优先级排序

从问题出发（发散收敛法，头脑风暴）

日常中会碰到大量的问题，由问题出发，通过归纳和总结是否存在同质性问题，从而一一化解。
分析思路从具体的需求出发，通过大量的体验来获得问题。将问题进行归纳、发散后整理需求为优化点。

  • 发散：通过大量的体验，记录所有碰到的问题，以及相应的解决方案。
  • 收敛：将已碰到的问题进行归纳总结，如“缺少曝光”
  • 再发散：将已归类的结果重新发散，看是否存在同质性问题，如“是否还有其他场景缺少曝光”，以及每个问题的解决放哪。
  • 再收敛：整理所有问题，思考所有问题中，是否有解决方案的通用性。并整理优先级，将当前最主要的需求作为突破点。

调研分析

调研分析是指通过寻找外部信息，来辅助个人决策的方式。分为以下三种调研方式

访谈调研

通过与需求相关人士进行访谈收集信息，有以下访谈的的方案

  1. 访谈范围
    1. 用户：需求的受众方，比如“竞猜用户”等。（注意：所有一线产品都需要有数个熟悉的用户，他们会帮你做决策）
    2. 运营等：需求的实际操作者，如运营，客服，商务等，往往他们离用户最贴近，最了解用户。可以问问运营线上的使用状况是否满意，是否符合运营推广基础；问问客服用户都会提出什么样的疑问，有没有什么常见的未满足的需求；问问商务当前是否有足够的曝光位，是否符合商务推广需要。
    3. 朋友：尽可能寻找更多人群征询意见。可以问问朋友对产品的第一印象如何，简单把玩后是否有体验不足之处；问问专业的朋友他们对该产品的看法以及独到的见解。
  2. 访谈渠道
    1. 即时沟通：适合与用户深度沟通，不尴尬又可持续的沟通方式。
    2. 电话沟通：适合广泛的向用户征询意见的用户调研方式。
    3. 当面沟通：适合与公司内的运营同事等，或者熟悉的朋友
  3. 访谈内容：
    1. 对产品的印象和使用问题：对所有人都适合那么问
    2. 对某个具体问题的看法：适合与熟悉的用户、同事、朋友沟通
    3. 对某件事情的反应：适合对用户，问用户在某种情况下，会做出什么样的结果，通过实际的结果来佐证自己的决策。
    4. 回忆性内容：适合对用户，根据用户的历史情况，询问为什么会在当时做出那样的决策。
  4. 访谈技巧
    1. 访谈概要：在访谈前需制定访谈计划，访谈计划包含以下内容
      1. 背景：为什么要进行这次访谈
      2. 目标：访谈想要达成什么样的结果
      3. 步骤：根据什么样的步骤进行访谈
      4. 内容：问什么样的问题，先问什么后问什么，若用户回答什么则需问什么等等
    2. 结果整理：对所有的访谈都需整理表格，分为数值整理和观点整理。
    3. 录音：若是电话沟通则需录音， 方便后续反复琢磨

竞品分析

竞品分析是指，通过竞品等其他产品的现状，来辅助个人决策的方式。
竞品分析分以下几个层次，此处引用用户体验要素的相关概念：

  1. 战略层：
    1. 整体体验产品，寻找产品最核心满足的一两个需求
    2. 用一句话概括产品的目标，并整理产品的背景、边界
  2. 范围层：
    1. 在产品所满足的所有需求中，选取最核心的需求。
    2. 主观判断：以“用户 - 特征 - 心理 - 场景 - 目标- 现有的问题 - 解决方案”的格式来表达每个需求的背景和方案（生成树状图）
    3. 客观分析：通过产品客观表现分析产品状态，如UGC更新频次表现用户活跃度等
  3. 结构层：
    1. 反复体验每个需求的流程，代入“用户 - 特征- 心理 - 场景”
    2. 体验完成目标的过程，记录下“不流畅”的点
    3. 评价该产品是否满足了核心需求，以及有哪些存在问题之处。
  4. 框架层：
    1. 整理该流程相关的所有页面
    2. 体验每个页面上任意按钮的点击，任意弹层的展开，任意文本的说明
    3. 整理所有界面中不合理之处
  5. 表现层：
    1. 看氛围：代入场景，看当前界面烘托分为是否符合目标受众
    2. 看风格：看界面中每个元素，是否都风格一致
    3. 看内容：代入场景，思考当打开该页面时，应当的阅读顺序，再反复体验，看是否界面的阅读顺序符合心理预期
    4. 看细节：细节上仔细观察，看是否存在逻辑硬伤

数据分析

数据分析是指，通过客观的数据表现，来辅助个人决策的方式
数据分析有以下内容

  1. 用以预测：通过竞品或当前相似业务的数据表现，预测另一业务的数据表现，如
    • 通过竞品每个主播视频观看量，以预测自己主播的视频观看量
    • 通过竞品每个UV创造的流水，以预测自己UV所创造的流水
  2. 用以数值设定：根据竞品数据或历史数据，调整数值设定，如
    • 通过收礼数、观看数、发言数的数值，调整人气值设定规则
    • 通过历史用户送礼情况，来调整会员的等级规则
  3. 用以判断：根据竞品或历史数据，帮助判断方案，如
    • 通过A/B两种方案的数据表现，用以决定最终方案
    • 通过当前付费额占比情况，用以决定是先做A，还是先做B
  4. 用以反馈：通过历史数据，反馈线上功能是否符合预期，如
    • 通过“高额用户续费率”，来反馈“贵族功能”是否符合预期
    • 通过“低额用户活跃程度”，来反馈“粉丝体系”是否符合预期
    • 通过“风控成功率”，来反馈当前风控机制是否合理

面试技巧及心得

发表于 2018-12-30
字数统计: 5.1k | 阅读时长 ≈ 17

面试前准备

  1. 进行深度的个人剖析
  2. 准备常见问题及回答
  3. 提早了解公司背景、理念、相关产品和JD
  4. 对产品所属用户群体及满足需求简单分析（试图站在对方的角度）
  5. 面试前联系HR说明已经到了

面试中技巧

  1. 隐晦透露出对公司产品理念的认同（挖掘产品优点）
  2. 面试时必须注视着对方的眼睛并时刻保持微笑和自信
  3. 回答问题时需停顿3秒
  4. 要通过逻辑性思维表达（首先、然后，第一、第二）
  5. 认同面试官提供的任何观点并作引申（是的，是的，并且）
  6. 面试官进入及离开时站立并握手
  7. 千万别忘记背景、目标、需求优先级

面试后表现

  1. 对HR表达感谢
  2. 如超过X天没有收到HR回复，则直接打电话询问

自我介绍

  1. 个人简单介绍
  2. 为什么要离开公司

面试官好，我是谢鑫。今天来面试一份产品经理的岗位。
本人毕业于浙江大学工业设计系，至今工作2年半。
大学时曾遭在网易云音乐实习产品策划，负责彩铃业务。独立接入三大运营商彩铃并推动业务上线。
毕业后加入了火猫直播，陪伴火猫直播从非常初创的阶段一直到现在，也算是经历了直播行业的变迁，从16年的千播大战到现在18年成熟玩家收获胜利果实。
在火猫初期的时候做基础体验项目，因为当时火猫立足未稳，功能不健全，需要完善产品体验以跟上第一梯队，为后期发展奠定基础。
后期行业更成熟了，强调变现能力，那现在我带领者3人产品团队负责增值项目，对营收负责。营收分为传统打赏类业务和竞猜类业务，传统打赏类业务针对主播内容的增值业务，在负责的Q2里，主播减少20%的情况下，保持收入略有上升。竞猜类业务是针对赛事内容，从零开始发展到每月数千万流水。
离开火猫直播是因为已经在此完整的度过了职业生涯第一阶段，产品经理期。而现在我需要寻求更大发展空间，需要在我更热爱的领域，和更好的团队做更令人兴奋的事情，而这些火猫无法提供。
来XX面试正是因为贵公司有着出色的产品和出色的团队，我们也有着相似的愿景和XXX的能力。我相信的加入相信也能为XX创造价值。

为什么加入火猫直播

因为我信奉那么几点：

  1. 产品经理的能力提升需要大量项目成本
  2. 产品助理+产品经理阶段培养流程大约2~3年
  3. 产品经理有部分“经验岗位”，需要优秀的导师培养
  4. 产品经理初阶能力和行业热爱无关，后期有关
    所以我选择了火猫直播，因为
  5. 火猫直播处于直播风口，有大量产品内容需完成，并且行业处于初期，相当于从零开始探索。
  6. 火猫有稳定的融资背景（华西村）能保证2~3年的稳定运营
  7. 火猫产品团队来源于腾讯（ceo、产品vp+产品总监），能提供较好的指导
  8. 个人还处于职业初期，有大量事情需要学习

而对于火猫直播而言

  1. 虽创业初期，但是有稳定发展打算。所以需要培养有潜力的新人作为未来主心骨。
  2. 新人如能快速成长，有足够的学习能力和可塑性，比招聘成熟的但思维较固化的员工未来更有有价值。
  3. 成熟的员工招聘太难，尤其是对于初创公司。
    这是火猫直播选择我的原因。

为什么离开火猫直播

先说明为什么加入火猫直播
然而现在情况不一样了

  1. 能力到达瓶颈，更深层次的能力需要更好的机遇和更多的热爱
  2. 衡量高级产品经理的指标不是个人能力，而是是否完成了出色的产品

你的产品最关注的指标是什么

不同的业务，以及不同业务不同阶段，最关注的指标会有所不同。

  1. 基础业务：用户观看时长 = 用户观看主播数 × 单个主播观看时长
    1. 产品：用户观看主播数，通过平台推荐机制让用户保持持续观看
    2. 运营：单个主播观看时长，通过主播签约，主播培养机制
  2. 打赏业务：收入 = 高付费用户收入 + 低付费用户收入
    1. 产品：低付费用户收入，通过产品引导和平台活动
    2. 运营：高付费用户收入，通过运营刺激和私人关系
      实际数据会根据情况而不同，如高付费用户收入，在缺少新增用户数，重点是“新增付费用户数”，在付费用户数量饱和时，是“付费用户续费率”
  3. 竞猜业务：用户投注流水 = 高付费用户流水 + 低付费用户流水
    1. 产品：低付费用户流水，通过产品引导和平台活动
    2. 运营：高付费用户流水：通过运营刺激和私人关系

你的职业规划是什么

  1. 阶段一：产品助理 + 产品经理期
    1. 目标：培养扎实的基础能力
      1. 通用能力：学习能力、执行力、沟通能力、心态、情商
      2. 专业知识：项目管理知识、技术理解
    2. 条件：
      1. 大量的项目经历
      2. 稳定的工作环境和工作范围（2~3年）
      3. 优秀的导师培养
  2. 阶段二：高级产品经理期
    1. 目标：将所学应用实际，负责独立产品业务线发展，培养行业全能力
      1. 专业技能：市场分析、市场营销、商务沟通、渠道管理……
    2. 条件：
      1. 出色的Leader及公司的支持
      2. 出色的团队
      3. 正确且个人热爱的行业
  3. 阶段三：产品总监期
    1. 目标：打造一款令人兴奋的产品，经历产品发展全流程
      1. 专业技能：财务管理、法务处理、市场前瞻性
      2. 团队影响力：方法论管理、知识传承、招聘
    2. 条件：
      1. 充足的资金
      2. 出色的公司Leader和投资人
      3. 正确且个人热爱的行业

你觉得你的优点有哪些（需有客观事实）

  1. 聪明（双商高，尤其是智商）
    产品经理每天会面临大量的，不可重复的问题，需要快速做出决策。这是难以批量化训练的。如果缺少足够的双商来灵活应对，工作中将疲于奔命。
  2. 出色的产品基础能力
    在火猫两年快速发展的工作经历了大量项目，并且都是核心业务。在产品基础能力如沟通能力、执行力、项目管理能力上，都有着扎实的能力基础。
  3. 丰富的行业经验
    经历了整个直播行业从初创，到风口，到衰落，最后胜利者收获果实。有着一些流量转化、用户留存、付费转化方面的经验。

你觉得你的缺点有哪些

  1. 粗心
    职业早期较为粗心。比如Leader交付10件事，最后可能会漏1件。处理方案是借助GTD工具，所有事情均罗列清单，定时检查以免遗漏。
  2. 没有丰富的大公司经验
    虽然我并不认为大公司培养一定比初创企业靠谱，但是的确缺少大公司经验，在流程组织，跨部门协调方面可能缺少指导。

为什么学物理的要学工业设计

  1. 为什么不学物理
    1. 数理化以锻炼思维为目的，是通俗性教育，并非需要投入专业。
    2. 物理竞猜是通往大学的门票，能得为什么不得，但并非真心热爱。
  2. 为什么是工业设计
    1. 所学专业和未来职业并非需要挂钩，但是通俗性能力任何职业都需学习
    2. 设计思维是通俗性能力，从小缺少思维锻炼，相当需要
    3. 因为热爱，热爱象形表达，热爱细致打磨的过程
  3. 学物理转工业设计对我带来了什么：较强的通俗性能力
    1. 理性思维：
      1. 产品的实践中不会有逻辑性错误
      2. 与技术的沟通中，不存在鸿沟
    2. 感性和创造力
      1. 产品有“个性”，且可以发挥创造力
      2. 与设计、运营类和沟通合作自如、

为什么学工业设计的要做产品经理

  1. 内容定义
    1. 明确我所学习的工业设计：以需求为导向，美学和功能为实现方式的产品设计。（并非美院类以美学为导向的产品设计）。涵盖了互联网应用设计。（也包含服务设计、硬件设计）
    2. 明确我们所理解的产品经理：以“互联网应用”为设计主体的产品经理。
  2. 共同点与差异
    1. 共同点：
      1. 同样的思维要求：以需求和问题为导向，兼而考虑工学、美学、经济学的应用
      2. 同样的使命感：让人和环境，人与人之间更融洽
      3. 同样的理念和准则：设计十诫、设计心理学
    2. 差异
      1. 媒介差异：互联网产品经理以“应用软件”为设计主体
      2. 流程化差异：拥有互联网特征，快速实践、资源整合、信息互动
  3. 有数理化和工业设计背景做产品经理更有优势：通俗性能力较强，既有技术性理解，又有发散性创造思维

你觉得产品经理要有哪些技能

  1. 【产品助理】工作0~0.5年
    1. 重点能力：流程意识（不做错事）、执行力
    2. 说明：产品研发过程中，都有合理的流程，保证良莠不齐的人力能产出质量合格的作品。产品助理若存在流程意识，则可合理地收到leader、项目经理等约束，保证产出结果不做错。
  2. 【产品经理】工作0.5~2年
    1. 重点能力：学习能力（做对事）、沟通能力
    2. 说明：该阶段产品往往是收到Leader所交付的需求，则需保证需求高效正确的完成。要求产品经理掌握扎实的基础功，如文档撰写，原型图绘制，数据分析等。并且该阶段开始负责产品的核心内容，需要与大量的运营、商务、财务的配合。要求产品经理能有足够的沟通能力，承担的起跨部门的沟通。并且能够各部门，各岗位的职责和技能，并给予积极的配合。
  3. 【高级产品经理】工作3~5年
    1. 重点能力：判断力（做对的事）、数据分析能力
    2. 说明：该阶段的产品经理，独立负责某一单一业务线，开始带领小产品团队去完成团队目标。需要产品有敏锐的市场判断力和足够的数据分析能力，能够制定团队目标和完成路径。以及初级的带领团队，影响图案鬼的能力。
  4. 【产品总监】工作5年~10年
    1. 重点能力：取舍&团队影响力（一起做对的事）、市场分析能力
    2. 说明：该阶段已经是产品总监，独立负责一款产品。需要产品有取舍，不仅知道要做什么，还要知道不做什么，以及优先级是什么。该阶段的产品带领了整个产品部门，需要能够有足够的团队影响力去推进全部门的进步。

你想加入的公司是怎么样的

  1. 最喜欢的公司分为
    1. 狼性：所有人都积极向上（不存在轻慢和效率低下的人）
    2. 团结：团队目标一致，没有内部斗争且能互相鼓励
    3. 优秀：个人能力优秀，都能独当一面
  2. 最喜欢的工作内容
    1. 工具性：符合个人爱好
    2. 个人为用户：自己就是用户，能够全情投入
    3. 初创阶段：行业内没垄断性巨头，且即将进入行业爆发期
    4. 用户基数大：能适用于广大的用户群，国民性产品
    5. 社会意义：能产生积极的社会效益，带来正能量

你做过最成功的产品是什么

增值业务的产品布局

  1. 简单介绍：
    1. 通过用户访谈和历史数据分析，获得了“高付费用户有付费瓶颈，且续费不稳定，但是付费比重极高。低付费用户活跃度低无法活跃氛围”的现状
    2. 策划了“打破付费瓶颈 -> 提高续费率 -> 提高用户活跃 -> 持续化运营活动”的产品策略
  2. 认为成功的原因：
    1. 对于公司：明显的收入增长，在直播逐渐流失的情况下
    2. 对于团队：恰如其分的发挥了产品和运营的作用，增强了团队凝聚力和作战能力。团队氛围Nice
    3. 对于个人：通过严谨有逻辑的推论得出产品布局逻辑，依赖于用户访谈和数据分析。加深了个人对于直播行业的理解
    4. 对于用户：获得了用户的好口碑。高付费用户获得更高的付费体验，并且乐于花更多钱；低付费用户在花钱不多的情况下产生了更高的活跃

你做过最失败的产品是什么

概率类小游戏。

  1. 介绍
    概率类小游戏是一种简单的流量变现的方式（近似于 收入 = 流水 × a）。就像是很多电影院里的的娃娃机一样。而直播则拥有足够的流量，且用户特征也与小游戏相符合。
  2. 方案对比：概率类小游戏的实现有两种方式：自研 或 接入第三方
    自研能做到和业务体系更好兼容，且无需和他人分成。缺点是开发成本大，项目上线周期长。
    外接需要与他人分成，但是可以快速上线，并且可上线多款。
  3. 初期方案选择自研的原因
    1. 认为上线一款小游戏就能带来大量流水：产品逻辑分析错误。
    2. 认为小游戏开发成本并不高：模仿他人界面即可
    3. 认为小游戏无需运营的参与，上线后就流水源源不断
  4. 初期方案选择的问题
    1. 高估了单一款小游戏带来的流水：小游戏新增和留存都极低，所以很依赖于新进流量。而直播流量是低新增高留存，而传统小游戏适用于高新增第留存。所以需要不断推出新款小游戏才能保证流水
    2. 低估了小游戏对设计、开发的高要求：小游戏对于视觉刺激要求极高，需要很高的设计要求和开发的即时性要求。
    3. 低估了小游戏运营在其中的作用：小游戏需要持续化运营，而并非业务展开后即有流水源源不断

说几个常用的产品，及其特征

  1. 轻芒阅读：
    1. 一句话描述：国内体验最好的RSS工具
    2. 优点：
      1. 交互细节：如阅读完文章以后向下滚动回到列表
      2. 置顶：让用户能区分“每天看的”和“偶尔看的”
      3. 信息流分类：让用户能区分时机，如晚上只看搞笑类的，工作只看工作类的
    3. 缺点：
      1. RSS本身的限制：门槛高，用户群体小（所以有轻芒杂志）
      2. 没有稍后阅读功能
  2. 即刻：
    1. 一句话描述：最细分主题的阅读工具
    2. 优点：
      1. 能找到独特的细分主题，而其他产品都无法提供
      2. 较“互联网”的人群分类，使得用户社区质量较高
    3. 缺点：
      1. 用户引导不强，大多数用户只会当成“今日头条”用，而无法发现其中“独特之处”，导致留存率低
      2. 缺少场景化阅读，是的阅读没有专注度
  3. 腾讯体育
    1. 一句话描述：赛事直播领域的典型
    2. 借鉴意义：对赛事流量和合理利用（高并发低留存产品）
      1. 独家版权，避免流量竞争，为后续功能作前置条件
      2. 推出一系列促进留存和日常打开频次的功能，如社区、资讯、视频
      3. 完善的赛事数据支持，体现专业性

怎么看待大厂和小厂的产品经理

这是一个我加入火猫直播时就一直在考虑的问题。
现在看来我的观点是这样的。

  1. 能力层面：分为两方面，毕业时的基础素质，毕业后的工作能力
    1. 毕业时的基础素质：
      1. 大厂同学经过了层层面试招聘，普遍基础素质要高于小厂。
      2. 但并不意味着小厂的同学基础素质一定差，只是没有“认证”而已。
      3. 对于我个人而言，算是半个大厂人。在本科期间经历了比较残酷的淘汰再“云音乐”这样一个互联网热门产品的产品实习，是当时唯一的产品实习生，也是云音乐当时唯一的本科实习生，可以一定程度上认证我个人本科期间的基础素质。
    2. 毕业后的工作能力：我认为和以下几点有关
      1. 是否经历了足够多的项目。大厂中的核心业务当然远好于小厂，但若是边缘业务则不如小厂的核心业务
      2. 是否有经验丰富的导师培养。大厂有着更优秀的导师和培养体系，但是势必培养会分散。如果碰到小厂中专心培养的导师也是很好
      3. 是否个人有足够的热情和适时的分析总结。取决于个人能力要求
      4. 对于我个人而言，再火猫一直负责最核心业务，并且陪伴火猫从小团队一直做大，经历了足够多的项目。产品总监和产品VP都是来源于腾讯的优秀员工，并且对我全力栽培。个人经常进行分析总结，有数百篇总结记录。
  2. 偏见层面：
    1. 大厂认证能够避免踩坑，能快速判断其能力层次。是员工中的硬通货。个人在招聘团队时也一样倾向于招聘大厂经历和高等院校。
    2. 小厂员工如若经历非专业面试官，可能会因为偏见而无法录取。但是越是专业的面试官越能透过虚名看核心能力。

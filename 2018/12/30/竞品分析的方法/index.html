竞品分析的方法

发表于 2018-12-30 | 分类于 职业技能
字数统计: 1k | 阅读时长 ≈ 3

定义

了解竞品都做了什么，为产品提供思路

  1. 正面：为什么他们能在行业立足/为什么他们会做成这样
  2. 反面：行业有什么弊端/功能有什么不足

原则

  1. 目标明确：明确本次竞品分析的目的，不做无休止的投入
  2. 由大到小：从源头开始分析（行业）而不单一的看产品和功能
  3. 客观真实：根据真实的数据结果来得出结论
  4. 分析结果：需有分析的结果产物，如行业结构图，功能发展图等

流程

  1. 明确分析内容：
    1. 内容：是行业了解类？还是功能调研类？
    2. 重点：如是调研“内容如何生产？”还是“界面如何使用”
  2. 整理分析步骤，罗列分析框架：
    1. 文档的格式：如背景、目标、行业情况、企业分布等
    2. 分析思路（调研方式）：如数据分析、用户反馈、行业报告等
  3. 寻找分析资料并整理归纳
  4. 进行思考输出结论和文档报告

内容

行业分析：

  1. 分析目标：
    1. 了解这个行业如何起源，都有哪些发展阶段？
    2. 从上游到下游，都有哪些类型的企业？分别依靠什么生存？
    3. 行业的企业之间，他们之间都有什么样的合作/竞争关系
    4. 行业整理发展情况如何？有什么用的数据表现？
    5. 行业未来可能的样子和方向？
  2. 资料来源：
    1. 行业专业性报告：企鹅智库、艾瑞数据、易观等
    2. 行业媒体&大V：如直播行业的“直播观察”、“今日网红”、“庄宇浩”等
    3. 网状搜索：从单个产品切入进行网状分析，如通过“斗鱼直播”的使用，发现“赛事举办方”、“经济公司”，再通过“赛事举办方”搜索了解其余“游戏公司”之间的关系等
  3. 分析结果输出：
    1. 行业企业分布
    2. 行业关系整理
    3. 行业大事记
    4. 行业简要数据情况
    5. 行业预测

企业/产品分析

  1. 分析目标
    1. 了解这些企业如何生存，做了什么产品？有什么样的功能？
    2. 各个企业有什么优势，有什么劣势？
  2. 资料来源：
    1. 企业官网/百科/微博/工商系统：企业自身的说明
    2. 企业财报：企业发展情况，收入来源等
    3. 媒体报告：媒体对企业的分析和观点
    4. 应用市场：企业的产品矩阵，每个产品的历史版本记录等
    5. 具体产品：企业具体产品的使用情况，产品布局等
  3. 分析结果输出：
    1. 企业概要说明
    2. 企业产品发展说明
    3. 企业收入情况
    4. 企业发展预测
    5. 企业特征分布 及横向评测

功能分析：

  1. 分析目标
    1. 功能由什么背景？为什么会做这个功能？
    2. 功能有什么使用场景？针对的用户群体是？
    3. 这些功能起到了什么样的作用，是拉新/留存/盈利？最终效果如何？
    4. 用户如何评价？还有什么改进之处？
    5. 各个不同产品之间，功能有何差异和区别？差异点原因是什么？
      任何程度的竞品分析，都需从源头（行业）开始，而不能单一的看该功能
  2. 资料来源
    1. 功能本身，根据使用流程和场景完整体验功能内容。
    2. 官网博客，官方对于该功能的理解和说明
    3. 应用市场，有关该功能的具体迭代记录
    4. 应用市场&论坛&贴吧&QQ群：用户对该功能的使用反馈
  3. 分析结果输出
    1. 功能发展报告：什么时候产生，每个版本迭代了什么内容
    2. 用户画像分析
    3. 功能数据分析报告
    4. 用户评价报告
    5. 不同产品的功能差异表和原因说明
    6. 功能的改进及优化意见

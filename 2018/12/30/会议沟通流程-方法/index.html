会议沟通流程&方法

发表于 2018-12-30 | 分类于 经验总结
字数统计: 1.4k 字 | 阅读时长 ≈ 6 分钟

原则

  1. 沟通尽可能多的人：让所有人都了解到自己所需要的信息，有以下人区分
    1. 周知：仅需要了解即可，无需参与，有问题事后补充，项目边缘人员
    2. 讨论：方案确定前需听取对方意见，但无需达成一致，项目相关人员
    3. 听取：方案必须与之讨论并达成一致，项目核心人员
  2. 沟通尽可能同步：当情况有不同时，需要传达
    1. 达成一致时：如会议结束
    2. 情况变更时：如出现问题时，或较长不沟通时
  3. 沟通尽可能及时：沟通越早越好，越快越好，反馈越及时越好
    1. 尽快发起沟通：当需要沟通时立即约定
    2. 尽快提供反馈：当有结果时立即同步
  4. 沟通尽可能高效：保证双方沟通只讨论未确定内容，不浪费沟通时间
    1. 约定会议时间、地点：无需临时有变更
    2. 提早发沟通内容：沟通只需沟通要点
    3. 表述精简：内容尽可能方便阅读
    4. 背景/目的思考清楚：沟通前明确背景、目的、内容等
    5. 方式高效：视情况采用 即时通讯、面对面、邮件、会议等方式
  5. 沟通需方式/流程合理：按照正确的流程和格式进行沟通，如以下情况
    1. 项目正式进度沟通：走项目邮件流程格式
    2. 项目即时需求沟通：项目组内即时通讯
    3. 非正式讨论：私下面对面沟通

场景

  1. 多向沟通（讨论）：需要复杂交流时，如请教问题、需求讨论、需求评审
  2. 单项沟通（讲述）：当信息不同步时单方面传递
  3. 单项沟通（聆听）：听取他人的讲述

方式

  1. 即时通讯：满足以下情况，使用即时通讯
    1. 非正式的周知性内容
    2. 短暂的讨论（3分钟内）
    3. 人数较少的沟通
  2. 会议：满足以下情况，使用会议
    1. 需要超过3分钟的沟通
    2. 较正式的沟通
    3. 参与人数较多的沟通
  3. 邮件：满足以下情况，使用邮件
    1. 较正式，低频次的沟通
    2. 作为备忘录，如进度周知等
    3. 参与人数较多的沟通

流程

  1. 沟通前：
    1. 明确会议背景、目的、内容、流程
    2. 明确会议参与人员、时间
    3. 明确会议成功条件/取消条件
    4. 参与者充分了解会议内容
  2. 沟通时：
    1. 约定进行会议记录的人
    2. 讲述会议背景、目的、内容流程、成功条件
    3. 开始会议正式内容
    4. 沟通快结束时与重复明确沟通结果
  3. 沟通后：
    1. 尽快（半小时要求）产出笔记周知群内
    2. 与大家确认是否有遗漏或错误
    3. 若正式纪要则需及时以邮件格式发出
    4. 会议内容一旦有结果/变更需及时周知

格式

  1. 问候：问候大家并简单陈述会议事项，如会议时间、会议名称
  2. 概要：以列表形式说明概要内容，需注意：
    1. 若需超过1分钟阅读，则需配以概要
    2. 概要需30秒内阅读完毕
  3. 正文：以图文形式说明正文内容（优先图片），需注意
    1. 若使用文本
      1. 单层列表若超过7项，则需再归纳
      2. 若与人相关，则需标注相关人员
      3. 若与时间相关，则需标注相关时间
      4. 若有重点，则需将重点标明
      5. 若使用附件，则需注明附件内容
    2. 若使用图片
      1. 图片需清晰明确
      2. 需标注想染他人注意的点
      3. 隐私信息需打马赛克，如数据、密码等
      4. 图标内容需有文本说明
  4. 总结性陈词：一般包含以下内容
    1. 邀请参与者阅读并回复纪要是否有误
    2. 接下来的计划安排，如下一次沟通的时间
  5. 附件：
    1. 文件命名必须包含【时间】【状态】【文件名】【发件人】【版本】
    2. 文件内容需注意格式统一规范，内容整洁。反复检查
    3. 尽量以Office或pdf格式发出
    4. 文件发出后需提醒对方已发出，请及时查收

附：会议纪要模板

感谢大家来参加本次XXXX会议，本次会议主要明确了以下内容
1、第一条的简要概述
2、第二条的简要概述
详情请见以下会议纪要：

【背景】
为什么要开会

【目标】
本次开会主要解决什么内容（如果简单可以和背景混在一起说）

【内容】
以列表的形式描述会议内容，如
一、版本内容确认1.权限管理。仅对部分时间段开启连麦，支持针对部分主播开放连麦；

【问题及跟进】
简要描述问题，并写明跟进人是谁、答复时间、答复形式，如
1、需与公会确认对于PK胜利一方主播的奖励（田由&郭宏鹏，10月15日下班前给出明确的奖励形式）

【后续】
后续的计划，如下一次会议会讨论什么内容，大概什么时候开始

如有遗漏或错误，请及时纠正和补充。会议文档/ppt将稍后发出，麻烦各位查阅。
如无问题，将会在今天下午15:00的时候给大家发总结邮件，请大家及时查收。

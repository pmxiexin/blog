个人管理/工作管理工具使用

发表于 2018-12-30 | 分类于 经验总结
字数统计: 2.1k | 阅读时长 ≈ 7

思维整理类

临时思绪/摘录- Unclutter&备忘录

  1. 场景：
    1. 有个转瞬即逝的想法；
    2. 突然发现一些有意思的点；
  2. 需求/Unclutter&备忘录优势：
    1. 能够最快速度的摘录；
    2. 能够在Mac上汇总并整理（iOS同步Mac）；
  3. 使用方式：
    1. 有想法/摘录时快速保存至Unclutter/备忘录
    2. 定时检查，处理内容

临时回复-Agenda

  1. 场景：
    1. 要大段回复他人时（会议纪要/总结/意见），如直接在发送框输入，容易误操作发送，需要先编辑好确认后，直接在发送框黏贴发送
    2. 会议纪要/总结/意见还有后续查询的意义，在通讯工具中容易丢失，需要有地方保存方便后续查看。
  2. 需求/Agenda优势：
    1. 能够快速调起/快速输入
    2. 有iOS/Mac端，能够做到内容云同步，并永久保存
    3. 便携的整理和阅读方式Org-Mode，能够快速阅读不同的条目
    4. 支持Markdown格式输入
  3. 使用方式：
    1. 当需要编辑大段文字时，先在Agenda中编辑
    2. 编辑完后粘贴至通讯工具中发送
    3. 当有需要查询历史信息时，能够通过搜索查询到

个人总结-Bear

  1. 场景：
    1. 有需要总结的工作内容
    2. 有点想法，想写内容
  2. 需求/Bear优势：
    1. 有iOS/Mac端，能够做到内容云同步，并永久保存
    2. 能够支持Markdown格式输入，并且写作体验足够好
    3. 能够支持多标签编辑的归类方式
  3. 使用方式：
    1. 将Unclutter/备忘录或其他渠道的写作想法，统一新建文章，录入标题并记为”待完成“
    2. 当有时间写作时，即打开”待完成“标签，依次根据内容完成。完成后去除”待完成“标签，并归类其他标签。

资料整理类

文件管理-Finder+Spotlight+Hazel

  1. 场景：
    1. 新资料：每日有各种各样的下载资源（主动下载/他人发送等），并且有些一下载完后必须马上打开
    2. 工作台：工作会产生大量的文件，并且需要快速寻找和打开
    3. 成果：工作后会沉淀大量文件，需要能够被快速搜索到
    4. 重要：有那么一批文件，每日都需要经常看情况和打开查阅
  2. 需求/Finder优势：
    1. 新资料
      1. 有一个文件夹，能够包括全部的新资料，而不同到处找
      2. 这个文件件能够被快速打开，而用通过很长渠道
      3. 新资料如果是doc等格式，能够默认打开，不然每次打开都很麻烦。
      4. 能够定期的清空/整理”新资料“，保证寻找资料更方便
    2. 工作台
      1. 最频繁的打开，能够被最快速度的访问和查找
      2. 一旦需要保存新文件，能够快速的找到该路径并保存
      3. 资料安全，能够被保存在云端
    3. 成果
      1. 能够快速辨识当前文件对应项目的状态（待评审/已上线）
      2. 能够先按照状态排序，再按照时间排序，方便查找项目
      3. 能够被快速搜索到文件
      4. 资料安全，能够被保存在云端
    4. 重要
      1. 这类文件夹能被快速的找到，因为经常访问
      2. 文件能被快速被归类为”重要“，但是不改变其本身的文件保存位置
      3. 有些类别的文件天生重要，可以自动归类
  3. 使用方式
    1. 新资料
      1. 将所有的能够创建文件的工具（浏览器/截图/通讯工具）的默认下载文件夹都选择为”下载文件夹“
      2. 将下载文件夹放置在底部工作条，能够快速打开查看
      3. 通过Hazel设置自动打开，如果格式为doc、dmg等格式，则下载完后自动打开。
      4. 每周设置提醒，清空并整理下载文件夹
    2. 工作台
      1. 将”桌面“设置为”工作台“文件夹，能够被最快速度的查找和访问
      2. 保存新文件时，选择”桌面“，最为快速
      3. 工作一旦完成后，就将工作文件进行整理保存在”文稿“中，方便查阅
      4. 通过iCloud，设置云端保存
    3. 成果
      1. 文件名格式为【状态】时间-项目-负责人，再通过文件名排序保存，可以使得先按照状态排序，再按照时间排序
      2. 当一个项目文件过多时，内分为小文件夹，方便查找
      3. 使用Spotlight进行文件搜索，更为快捷
      4. 通过iCloud，设置云端保存
    4. 重要
      1. 将”重要“设定为”标签“，能够不改变文件保存位置
      2. 将”重要“放置在底部条上，能够快速打开和访问
      3. 通过Hazel，将部分状态的文件自动设定”重要“标签

图片整理-Eagle+Finder

  1. 场景：
    1. 设计确认设计稿
    2. 突发的灵感图（产品使用中借鉴之处，竞品调研图）
  2. 需求/Eagle优势：
    1. 能够云端保存，不会轻易丢失
    2. 方便查找和整理
  3. 使用方式
    1. 与个人相关设计项目，当设计稿确认后，将其归类到Finder，对应的文件夹中
    2. 产品使用中，突发的灵感图和借鉴图，及时复制到Eagle中保存
    3. 定期的进行图片整理，如有灵感可整理到Bear货需求list中

工作管理类

任务管理-OmniFocus

  1. 场景
    1. 每日的日常任务，需要经常提醒自己要完成
    2. 他人提供的小任务，需要有记录，防止以后忘记
  2. 需求/OmniFocus优势：
    1. 能够设置重复任务，每天都能提醒自己该做什么，养成习惯
    2. 能够快速记录他人提供的临时任务，并且查阅检查
    3. 能够云同步，在各个端都能检查当前的情况
  3. 使用方式
    1. 建立重复项目（每天重复、每周重复）能够在合适的时候提醒自己，现在应该做什么
    2. 当收到他人的需求时，如能快速解决则马上解决，否则列入收件箱中。
    3. 每天中午&下班前整理收件箱，若能快速解决则马上解决。否则列入个人每日项目中并设置提醒时间。
    4. 每天上班对个人的任务清单做查询，确认今天的工作内容；每天结束时作校验，确认项目的进度情况

邮件管理-网易邮箱大师

  1. 场景
    1. 收到他人邮件，需进行归类和重要邮件的标注
    2. 发邮件时，需要按照指定的格式发出
    3. 工作离职后，邮件内容还有大量信息，值得未来借鉴
  2. 需求/网易邮箱大师优势
    1. 能够有自定义规则，来进行邮件的整理和归类
    2. 能够有旗标功能，进行邮件的快速标记
    3. 发邮件时，能够进行辅助，快速生成格式邮件
    4. 能够进行邮件的导出和保存，使得永久保存
  3. 使用方式：
    1. 对邮件进行规则设置，指定规则的邮件移动到指定分类
    2. 对重要的邮件打上旗标
    3. 设置邮件模板，当发邮件/回复时根据模板自动生成邮件内容
    4. 离职时对邮件进行导出保存

项目管理-OmniPlan

  1. 场景
    1. 产品每个人的项目管理，每周要确认下周的工作内容
    2. 开发排期确定后，需要能定期检查当前项目的情况（进行中，已结束）
    3. 给老板反馈进度时，需能结构化输出结论
  2. 需求/OmniPlan优势
    1. 能够有甘特图，以项目为维度检查当前进展情况
    2. 能够有人力图：以人为单位检查每个人的工作内容
    3. 能够对日历进行同步：每天提醒今日的进展情况
    4. 能够云同步，在各个端都能检查当前的情况
  3. 使用方式
    1. 当增减新项目时，在其中新建项目，并排列项目优先级、关联关系等情况
    2. 当需求排序确定时，在其中分配每个开发的排期情况
    3. 将OmniPlan与日历同步，每天能检查当前所有项目的开发情况

如何进行项目立项分析

发表于 2019-01-28 | 分类于 职业技能
字数统计: 1.1k | 阅读时长 ≈ 3 分钟

为什么要项目立项分析

怎么样少做傻逼需求？——必须要在项目立项时就遏制住。
合理的团队应有合理的项目立项流程。领导层决定整体的项目走向，总监层决定具体项目的安排。而立项分析一般会交给负责该业务的产品经理。任何的项目立项都需谨慎，且必须由产品、技术、运营团队一致通过。好的项目立项能够避免产品走弯路，提升团队的工作效率和主人翁精神。
一个我觉得还不错项目立项流程可以参见我《关于项目流程的思考》，而产品怎么写出一个好的项目立项分析可以参见本文。

什么是好的项目立项分析

一个好的项目立项分析需至少回答这些问题：

  1. 为什么要做这个项目？（碰到了什么问题。想完成什么目标。）
  2. 别人是怎么做的？（竞品的方案是啥）
  3. 打算怎么做？（我们可能的方案是啥。打算什么时候做好）
  4. 项目成果会怎么样？（数据会有什么变化呢）
    项目立项分析是一件非常考验产品经理逻辑推导、资料收集、数据分析能力的技能。

分析的流程

明确目标

和上级明确本需求的目标，大家保持共识。避免后续出现理解的偏差。
明确目标至少需包含以下内容：

  1. 主要目标、次要目标、小目标等。以及明确各个目标之间的优先级
  2. 当前现状，以及业内水平。如目标为提升付费率，则需收集当前付费率情况，以及业内同类型产品的付费率情况。
  3. 可接受的付出的成本。包括资源成本和人力工时。

用户需求分析

虽然目前大多数产品通过“参考”就可以完成需求。
但是作为一个古典主义的产品经理，还是要倡导大家，至少要有过自己独立的，用户需求分析的过程。
用户需求分析至少需包含以下内容：

  1. 针对什么样的用户群？他们分别有什么样的特征？
  2. 他们分别有什么样的场景，并且在各个场景下，利用他们的什么需求来做功能？
  3. 最终有哪些功能可以帮助实现目标，以及优先级是什么样的。
    详情可见我《如何改进/分析产品》

竞品分析

通过竞品分析，得知业内产品对于该方法的处理方式，从而指导自己的方案。
一般来说，大多数产品没有强到可以自己从头开始构想，能够找模作样的实现业内领先产品的方案，并且进行小范围的微创新已经很了不起了。
竞品分析至少需包含这些内容：

  1. 业内各个产品的处理方式，及背后的原因分析。
  2. 各个产品之间的共性、差异性。可归纳为几种基本的方案。
  3. 有没有从这些方案衍生出的更佳方案。
  4. 哪些方案适用于我们，哪些方案不适合我们。以及我们对方案的优先级判断。
    详情可见我《竞品分析的方法》

期望实现方案

根据用户需求分析，可以获得“符合用户需求的功能list”；根据竞品分析，可以获得“竞品参考功能list”。将两者整合在一起，并判断功能的优先级，并根据成本要求分多版本计划排期。
选择首个版本的内容，将方案进行细化输出。
期望实现方案需包含以下内容：

  1. 一共分X个版本，每个版本实现的内容。
  2. 首个版本的需求方案简要描述。
    关于优先级的排序可见我《如何判断需求优先级》

项目预期

需包含以下内容：

  1. 预期的时间安排，人力安排，资源安排。
  2. 预期的实现效果，数据结果
    关于预期的实现效果可见我《产品经理的数据分析方法》

产品中间件

发表于 2019-01-27 | 分类于 职业思考
字数统计: 1k | 阅读时长 ≈ 3 分钟

中间件。这词一般程序猿GG会比较爱说。就是他们口中“不要重复造轮子”中的那个“轮子”。一般指一些有着自己独特特性的小功能的集合。比如有个中间件叫“读写文件”，那以后程序猿GG一旦碰到需要读写程序，只要使用这个中间件即可，不需要自己再去了解底层的读写逻辑了。

同样，在产品功能中，也有那么一些“中间件”。使得目前大多数产品不需要去研究最基础的功能实现，只需要想怎么使用就可以了。
比如如果想提升用户忠诚度，产品无需完完全全构思一个功能。只需要考虑到，一般提升用户忠诚度的中间件有“等级系统”、“积分系统”。然后再阅读这两个中间件的使用说明书，使用说明书上会说明这个中间件有什么用的前提，适用于什么样的场景，有什么样的注意事项，怎么一步步接入等。（欢迎阅读我对等级的思考，本站搜索“等级”、“积分”即可）然后产品自己配制下产品参数，把这个中间件像插插座一样插入自己的产品就可以了。

常见的一些中间件：

  1. 产品基础功能：基本每个产品都会碰到，往往有可靠的第三方提供
    1. 账号体系：用户登录/ 注册/ 绑定手机/ 找回密码/ 上传头像 等等
    2. 推送系统：将消息推送到用户手机上（个推）
    3. 广告系统：后台运营配置，前端显示广告（百度联盟，Google Ads）
    4. 分享系统：将页面分享到社交网络（友盟）
    5. 统计系统：统计应用数据表现（友盟、神策、Growing IO）
    6. 支付系统：用户通过第三方渠道付款购买服务（支付宝、微信支付、IAP）
    7. 反欺诈系统：帮助产品过滤脏词&黑产，如文本过滤等（数美、极验）
    8. ……
  2. 行业解决方案功能：行业性通用产品功能，往往也有可靠第三方提供
    1. CDN系统：将文件/ 视频/ 图片 通过CDN加速让用户快速访问（七牛）
    2. 直播系统：从主播推流到用户播放的一套解决方案（腾讯云、网宿）
    3. 点播系统：点播视频的播放（腾讯云、网宿）
    4. ……
  3. 运营方式：通用&固定的运营方案模板
    1. 用户忠诚度方案：
      1. 任务/积分系统：用户完成指定行为可以获得积分，积分能兑换奖励
      2. 等级系统：用户完成指定行为可以提升经验值，经验值能获得权益
      3. ……
    2. 收入提升方案（详情可见运营常见策略）：
      1. 满减：达到一定的条件，可以减少一定金额，往往用于拉单笔消费额
      2. 满送：达到一定条件，可以赠送东西，往往用于清库存
      3. 降价：价格降低，往往用于清库存
      4. 免费：免费赠送，往往用于拉新，或是其他方式盈利
      5. 预付：预先付一笔高额支出，可以获得额外奖励，往往用于拉留存
      6. ……
    3. 活动运营方案：
      1. 排行榜：用户互相竞争排行，排行榜头部可获得奖励。往往用于拉收入，一般用于用户分化明显的业务（如打赏、精彩）
      2. 邀请返利：邀请他人参与，双方可获得收益。往往用于拉新。
      3. 组队打怪：多人组成小队，一起参与活动。往往用于促活。
      4. ……

中间件的存在，使得产品能够不断的复用功能，提高工作效率，将时间专注于真正的用户需求上去。未来也会有更多关于产品中间件的分析（我也会多写一些），让更多产品能够方便地使用这种工具。
但中间件的使用，只能作为常规的增长方式，如果需要有突破性的进展，同样需要突破性的创新。

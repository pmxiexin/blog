积分的意义与设定方式

发表于 2018-12-30 | 分类于 经验总结
字数统计: 2.6k 字 | 阅读时长 ≈ 13 分钟

积分的定义

用户通过完成指定行为“获得积分”，积分可“兑换奖励”。

积分的作用

积分是促活/营收型产品功能

  1. 对于用户：
    1. 获得收益：完成指定行为后，可获得收益
  2. 对于平台：
    1. 核心——刺激使用：在获得积分时引导用户多完成指定行为；在兑换奖励时引导用户多获得积分从而二次消费
    2. 提升用户离开成本：若用户积分较高则难以离开

积分的类型

返利型积分

特征

  1. 一般为电商型产品使用，目标为GMV，如飞猪、京东。尤其是同质化严重的传统行业（航空/酒店）
  2. 当用户消费一定金额后，会将支付金额的一定比例以积分的形式返还给用户。而用户通过积分在二次消费时能获得减免，或者直接兑换奖励。
  3. 目标是用户作同质性商品的购买决策时，能够受到积分引导。
  4. 奖励一般都有市场价值。

前提

  1. 用户和平台都能算得积分收益比（返利比），并计入成本支出
  2. 积分是当用户“选择两难”时帮助用户决策，前提是能与竞品能提供同质性产品，如产品差异过大则积分无价值
  3. 产品本身是否存在二次消费必要，如低频的需求如买房、买车等不存在二次消费，则积分的价值对用户是累赘，直接赠送东西更简单有效。
  4. 当前促活、营收是最重要的事情。获客成本较高时，以及留存率稳定时。如当获客成本极低时，尽可能获得更多新用户才是最值得的事情。

流程

  1. 明确积分的目的：一般返利型积分都是以消费额作为目标
  2. 确定积分来源及价值：
    1. 确定来源功能：一般返利型积分都是以消费订单作为积分来源。
    2. 确定返利比例，如每100元订单，可以返还给用户多少钱。计算方式如下：
      1. 计算产品收益率。如用户购买每100元商品，可收益10元，收益率为10%
      2. 计划用户运营总支出，如每收益10元，都拿5元来做用户运营（往往和当前市场环境有关，可参考竞品用户运营支出）
      3. 计划积分占用户运营总支出的比例，如每5元用户运营，其中1元作为用户等级奖励，1元作为用户任务奖励，1元作为渠道推广成本，剩下2元作为积分返利成本。
      4. 则返利比例为，每100元，返利给用户2元。
    3. 确定积分价值，如1元=100积分。一般有以下考虑
      1. 和客单价有关。主流订单的积分能够在十/百位数，能够让用户在下单时获得“满足感”，如酒店客单价在200~300元，能获得20~200积分，都较为合适，而如果只给2积分，则用户“感觉上收获很少”
      2. 和订单下限金额有关，需保证尽可能多的订单，都能够获得积分。如电商有50元左右订单，如无法获得积分，则会给用户负反馈。
      3. 积分除返利外还有其他渠道。如积分需应用于签到、任务等，则积分价值需能匹配任务价值。如若1积分=10元，则签到、任务等获取积分的价值则过大。
    4. 根据当前用户行为统计（如没有则可采用行业数据），经过一定”预估加成“后对用户积分值进行估算，统计用户积分值的分布。
    5. 检查是否存在”超出预期范围“的积分值，如产品中心化明显，则中心用户则能获得最大化积分。可进行积分限制，如每天最多能获得的积分来限制用户积分值。
  3. 确定积分出口：返利型积分都可直接抵换平台货币，或是兑换实体奖励
  4. 检查安全漏洞：检查是否有被黑产恶意刷积分的可能，一般有以下方式避免：
    1. 增加验证码：在重要环节中如“领取奖励”时，设置完成验证码后方可领取。
    2. 用户限制：限制用户身份，如“已绑定手机”、“已连续登陆X天”等
    3. 唯一性限制：同一手机、设备号、ip进行限制，每一项只允许领取X次
    4. 最大损失：每天/总共设置领取奖励范围，如单日最多100个奖励
    5. 第三方校验：接入数美/极验等第三方黑产校验服务
    6. 定时检查：定时获取参与用户信息，观察是否有存在着黑产的可能

鼓励型积分

特征

  1. 一般为社区型产品使用，目标为DAU，如什么值得买。
  2. 当用户完成指定行为如签到、发动态等（一般有次数限制），就会获得一定的积分奖励。积分奖励可兑换奖品
  3. 目标是让用户能够更好融入社区，培养社区习惯。
  4. 一般为平台免费的奖励，如抵用券，身份标识等

前提

  1. 产品存在明确的，少量的，关键性的指标和用户行为。
  2. 已验证产品对用户存在实质性价值，否则即使通过积分鼓励也效果有限
  3. 当前促活、营收是最重要的事情。获客成本较高时，以及留存率稳定时。如当获客成本极低时，尽可能获得更多新用户才是最值得的事情。

流程

  1. 明确积分目的，是提升打开频次？用户活跃程度？还是……
  2. 明确积分来源及数值：
    1. 确定来源行为：来源行为主要和KPI数值有关
      1. 如KPI为“打开频次”，则发放积分行为则为”每天首次打开App“等
      2. 如KPI为”拉新用户数“，则发放机会行为为”每日分享“等。
      3. 注意：需考虑积分鼓励下的行为，可能带来的副作用。如鼓励用户”每日发言“，则可能用户每天发”垃圾话“从而破坏社区氛围
    2. 确定行为权重：如签到是1单位积分，而分享为5单位积分，比较内容一般为：
      1. 操作的难易程度：越是复杂的操作，会收获更多的积分
      2. 目标的重要程度：如产品DAU为KPI，则应给签到更多积分
      3. 【核心】行为带来的利润收益：如用户操作1个行为能带来更多收益，则应收获更多收益。简要计算方式：每次行为的收益=新增行为的收益/总行为次数。举例如下：
        1. ”签到积分“行为收益
          1. 新增行为的收益：1个用户平均每天浏览10条广告，可获得10元。”签到积分“使每10个老用户，新增1次登录。则 ”签到积分“收益为：每10个老用户，可新增收益10元。
          2. 总行为次数：每10个老用户，总共5次登录。共发放5次”签到积分”
          3. 每次“签到”的收益为2元（10/5）
        2. “分享”行为收益
          1. 新增行为的收益：如每个用户的终身利润为50元。分享积分“使得每10个用户，可新增1个新用户。则“分享积分”的收益为：每10个用户，可新增收益50元。
          2. 总行为次数：每10个用户，总共5次分享，共发放5次“分享积分”
          3. 则”分享“带来的收益为10元（50/5)。
      4. 确定积分数值及”积分—金额比例“：
        1. 将权重最小的行为积分设定为1或10，如签到权重最小（为2），设定为10积分。
        2. 其他积分按照权重比例依次放大，如”分享”权重为10，则为50积分。
        3. ”积分—金额“比例也由”行为收益“确定，如”签到“是10积分，可收益2元，则”1积分=0.2元“
      5. 用户积分统计：根据当前用户行为统计（如没有则可采用行业数据），经过一定”预估加成“后对用户积分值进行估算，统计用户积分值的分布。
      6. 检查是否存在”超出预期范围“的积分值：如产品中心化明显，则中心用户则能获得最大化积分。可进行积分限制，如每天最多能获得的积分来限制用户积分值。
  3. 确定积分出口：鼓励型积分的出口，以平台特权和免费的稀缺资源为主。
  4. 检查安全性漏洞，同上文所描述。

其他积分类型

奖励型积分

  1. 一般为”拉新阶段产品“使用，目标为新用户，如人人贷
  2. 当用户进入产品时，免费赠送积分，用户可拿积分直接获得奖励。而奖励是另一产品的拉新方式。
  3. 用户会因获得奖励后的”损失厌恶“，从而领取奖励并使用。奖励一般是新用户特权，如新用户满减券等。

留存型积分

  1. 一般为”留存阶段产品“使用，目标是用户能够多待一会。往往快消型产品需要，如”趣头条“、”火山小视频“，经常被包装成”红包“
  2. 当用户使用产品后自动获得积分，但积分不能立即使用，有冻结期或是需达到一定程序才能使用。
  3. 用户会因为舍弃账号后的”损失厌恶“，继续使用产品。

其他说明

积分是否需定时清零：

一般清零型积分有以下考虑

  1. 财务需要：返利型积分都有实质性价值，而对财务来说则是负债，需每年清算。如有有效期（如每年失效）则方便财务清算。
  2. 积分价值变化：如原定1积分=0.1元，而新的功能和任务产生，或是市场环境的变化，导致积分价值过高/过低。则需要定时清零来保证积分价值的稳定。
